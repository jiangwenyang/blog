浏览器事件机制

August 30, 2018

学习 JavaScript 都知道，浏览器的事件触发分为三个阶段。

  1. 从最外层容器（通常是 window）向事件触发区域传播。— 捕获阶段（如果注册了捕获事件，将在这个阶段触发）
  2. 传播到事件触发区域 — 目标阶段 （触发注册事件）
  3. 从触发区域向 window 传播 — 冒泡阶段 （如果注册了冒泡事件，将在这个阶段触发）

事件注册

事件注册分为 3 种（其实是四种），如果包括下面这种最最古老的写法：

<div onclick="handleClick()"></div>

<srcipt>
  function handleClick(){ }
</srcipt>

不过这种和下面第一种类似（都是上古时期的写法），就归为一种吧。

  1. 传统的事件注册,在元素上使用事件属性。

    document.getElementById("myDiv").onclick = function() {};

    优点：兼容所有浏览器

    缺点：只能注册一个事件处理器（类似于给 onclick 重新赋值），只支持事件冒泡

  2. W3C 标准事件注册EventTarget.addEventListener()

    参数：

    • type 事件名（不带 on 的）
    • listener （文档说实现了 EventListener 接口的对象，或者是一个函数，其实具体使用上就是一个函数）
    • [options] 一个对象，指定 listener 的一些行为
    • [useCapture] 布尔值，默认 false，为 true 时表面事件在捕获阶段触发，否则在冒泡阶段触发

    [MDN 文档]: https://developer.mozilla.org/zh-CN/docs/Web/API/EventTarget/attachEvent

    var myDiv = document.getElementById("myDiv");

myDiv.addEventListener("click", function() {}, true);

myDiv.addEventListener("click", function() {}, false);

myDiv.addEventListener("click", function() {}, {
 capture: true,
 once: true
});

    优点：同一个事件可以注册多个处理器，按照注册先后顺序执行，支持自定义事件。

    缺点：不兼容 IE6-IE8，IE9 及之后 IE 浏览器可用

  3. IE 家族的EventTarge.attachEvent() (IE6-IE10 可用，但是 IE9 就已经可以用 addEventListener 了，直到 IE11 才不支持 attachEvent)

    参数：

    • eventNameWithOn 以 on 作为开头的事件名称，例如”onclick”。
    • callback 执行的回调（不接受 event 参数，可以在 window.event 对象上获取）

    缺点：不支持事件捕获，不支持自定义事件。

    优点：在不支持addEventListener()的老 IE 中可以为一个事件绑定多个处理器（执行顺序时从后到先的顺序）。

    var myDiv = document.getElementById("myDiv");
myDiv.attachEvent("onclick", function() {});

传统事件绑定、EventTarget.addEventListener() 和EventTarge.attachEvent()还有个区别在于事件处理函数中的 this 指向，前两者指向绑定事件的 DOM 元素对象，后者始终指向 window 对象。

事件注册的兼容性写法（<=IE8 的 IE 浏览器不支持事件冒泡，可以通过添加 polyfill 的方式兼容）：

if (window.addEventListener) {
  object.addEventListener("click", function() {}, false);
} else if (window.attachEvent) {
  object.attachEvent("onclick", function() {});
} else {
  object.onclick = function() {};
}

事件取消

对应三种事件注册，事件取消也有三种：

  1. 取消传统事件绑定，将对应元素上的事件属性赋值为 null。

    var myDiv = document.getElementById("myDiv");
myDiv.onclick = function() {
 alert("myDiv");
};
myDiv.onclick = null;

  2. W3C 的EventTarget.removeEventListener()取消使用EventTarget.addEventListener()绑定的事件。

    参数：

    • type
    • listener （必须和EventTarget.addEventListener()指向同一个）
    • [useCapture] 取消的事件处理是否为捕获阶段，默认为 false，true 对应捕获阶段，false 对应冒泡阶段。统一个 listener 如果分别注册在捕获阶段和冒泡阶段，则可以分别取消，互不影响。
    const myDiv = document.getElementById("myDiv");
function a() {
 alert("a");
}
myDiv.addEventListener("click", a);
myDiv.removeEventListener("click", a);

  3. IE 家族的EventTarge.detachEvent() (IE6-IE10，IE11 不再支持)

    参数：

    • eventNameWithOn 以 on 作为开头的事件名称，例如”onclick”。
    • callback 执行的回调
    var myDiv = document.getElementById("myDiv");
function a() {
 alert("a");
}
myDiv.attachEvent("onclick", a);
myDiv.detachEvent("onclick", a);

阻止事件传播

  • W3C 标准

    使用event.stopPropagation()阻止事件传播（不仅仅可以阻止事件冒泡，也可以阻止其后的事件捕获）

    还有个不常用的event.stopImmediatePropagation()，不仅仅能阻止事件传播（捕获和冒泡），还能阻止目标阶段执行其他注册事件（例如同时注册了多个 click 事件，则在某个事件函数中执行到event.stopImmediatePropagation()其后的事件函数将都不再执行）。

  • IE（IE6-IE8）

    使用event.cancelBubble=true;阻止事件冒泡。（IE6-IE8 不支持事件捕获，因此没有阻止事件捕获的方法，都不支持还阻止个毛线……）

阻止默认事件

阻止如<a>标签点击后的默认跳转事件。

  • W3C 标准

    event.preventDefault()

  • IE (IE6-IE8)

    event.returnValue=false;

  • 传统事件绑定

    使用下面这两种古老写法的事件绑定的，直接在事件处理函数中return false:

    最老的写法：

    <div onclick="handleClick()"></div>

<srcipt>
function handleClick(){ // dosomthing return false; }
</srcipt>

    也很老的写法：

    var myDiv = document.getElementById("myDiv");
myDiv.onclick = function() {
// dosomthing
return false;
};
